# 宠物互动 App PRD（补充提纲）

## 1. 产品定位

围绕"**宠物陪伴 + 运动成长 + 轻联机互动**"的 iPhone App。用户登录后获得一只默认猫咪作为宠物，猫咪根据用户现实中的运动状态切换展示动作；用户通过步行、跑步获得步数；主界面存在一个定时解锁的宝箱，用户可消耗步数开启宝箱并获得装扮；用户还能进入房间与其他玩家进行轻量实时互动，并发送系统表情。

权威来源：`docs/宠物互动App_总体架构设计.md` §2 + `docs/宠物互动App_MVP节点规划与里程碑.md` §1。

## 2. 目标用户

- 有走路 / 跑步习惯的 iPhone 用户
- 喜欢萌宠陪伴、装扮养成类轻度互动的玩家
- 接受轻社交（4 人房间 + 系统表情），但不希望被复杂权限管理打扰

## 3. 核心价值

- **运动 → 资产**：把日常步数转化成宝箱开启资格与装扮收益
- **宠物陪伴**：猫咪状态贴合用户的真实运动节奏
- **轻联机**：4 人房间内即时看到他人猫咪状态 + 表情广播
- **可收集装扮 + 合成进阶**：实例化道具体系，10 个同品质实例可合成 1 个高阶实例

## 4. MVP 范围（已确认）

### 必做
- 游客登录
- 默认猫咪
- 首页展示
- 步数同步
- 宝箱倒计时与开箱
- 装扮实例化背包（仅展示）
- 穿戴 / 卸下（缺图时文字降级渲染，确保独立可验收）
- 装扮渲染配置（per-item render_config，让穿在身上的样子可定制）
- 手动选材合成
- 房间创建 / 加入 / 退出
- 房间内发表情
- 分享房间链接

### 暂不做
- 复杂好友系统
- 房间聊天
- 排行榜
- 交易系统
- 多宠物体系
- 工会 / 社区
- 微信绑定（结构预留，节点 2 不实现 UI）
- watchOS 端（后续阶段，本文档不涵盖）

## 5. 关键产品规则（已确认）

| 规则 | 描述 |
|---|---|
| 账号 | 默认游客登录（无登录页）；后续支持微信绑定；同一真实用户始终对应一个稳定 `user_id` |
| 步数 | 读取 iPhone 系统统计步数；步行与跑步收益一致；步数既是成长记录也是可消耗资源 |
| 宝箱 | 同时只存在 1 个；倒计时 10 分钟；开启固定消耗 **1000 步**；可开启状态可一直保留至开启 |
| 装扮 | 实例化数据，每件实例都有唯一 id；同一种装扮配置可被持有多件 |
| 合成 | 玩家手动选 **10 个同品质实例** → 1 个高一阶品质随机装扮实例；不要求同部位 / 同配置 id；legendary 不可续合 |
| 房间 | 最多 4 人；同时只能加入 1 个；可主动退出；不能踢人；只要有人在就不解散 |
| 表情 | 系统固定配置，仅作房间内广播提示，不改变宠物属性 |

## 6. 12 节点 MVP 推进顺序

| 节点 | 名称 | 核心目标 |
|---|---|---|
| 1 | App 与 Server 可运行 | 打通最小工程与 ping 联调 |
| 2 | 默认游客登录 | 实现无登录页自动游客登录 |
| 3 | 小猫动作与步数上报 | 本地猫动作表现 + 服务端步数接收 |
| 4 | 基础房间功能 | 创建、加入、退出、成员进出通知 |
| 5 | 房间内猫动作同步 | 房间成员可看到其他人的猫动作状态 |
| 6 | 房间内表情发送 | 点击自己的猫发送表情并广播给他人 |
| 7 | 宝箱倒计时与奖励弹窗 | 倒计时、1000 步开箱、奖励展示、道具配置 |
| 8 | 仓库功能 | 奖励入仓并可查看拥有的道具（仅展示） |
| **9** | **穿戴 / 卸下** | 玩家从仓库选择实例穿戴；缺图时以文字代替图像 |
| **10** | **装扮渲染配置** | server 提供 per-item render_config；client 按参数渲染 |
| 11 | 合成功能 | 手动选择 10 个同品质实例合成更高品质道具 |
| 12 | 分享房间链接 | 生成房间链接并支持通过链接加入 |

依赖关系：

```
节点1 -> 节点2 -> 节点3
节点3 -> 节点4 -> 节点5 -> 节点6
节点3 -> 节点7 -> 节点8 -> 节点9 -> 节点10
节点8 -> 节点11    （合成 不依赖 穿戴 / 渲染配置，可与之并行）
节点4 -> 节点12
```

详见 `docs/宠物互动App_MVP节点规划与里程碑.md` §3-§8。

## 7. 技术总览

- **客户端**：iPhone，Swift + SwiftUI，MVVM + UseCase + Repository
- **后端**：Go，模块化单体（**不**拆微服务），Gin + GORM/sqlx
- **主存储**：MySQL 8.0
- **缓存与实时态**：Redis
- **配置**：YAML + 环境变量覆盖
- **协议**：REST + WebSocket
- **三端独立目录**：`server/` (Go) / `ios/` (Swift) / `watch/` (watchOS，暂不考虑)
- **跨端契约**：通过 `docs/宠物互动App_V1接口设计.md` 同步，不通过共享代码

详细架构、模块边界、接口、表结构、时序见 `docs/宠物互动App_*.md` 七份设计文档。

## 8. 交付与验收原则

- **节点顺序不可跳**：每个节点完成后才进入下一个
- **同一节点的 iPhone 端 + Server 端拆独立 Epic**，每节点末尾设"对齐 Epic"做联调验收
- **资产类操作必须事务**（开箱、合成、穿戴、加入房间、游客登录初始化）
- **状态以 Server 为权威**（步数余额 / 宝箱状态 / 背包归属 / 合成结果 / 房间成员关系）
- **server 测试自包含**，不依赖 iOS / watchOS 真机；真机联调放对齐 Epic
