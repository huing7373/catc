## Story

As an iPhone 用户,
I want App 启动后调 `POST /api/v1/auth/guest-login` 带本机 Keychain 的 `guestUid` 拿到一个稳定的 JWT token + 用户身份 + 默认猫，再次启动复用同一身份不重复创建,
so that 客户端 0 操作进主界面（无登录页 / 无注册页）；server 端在**首次** guestUid 落库时把 `users / user_auth_bindings / pets / user_step_accounts / user_chests` 五张表的初始行**在同一 MySQL 事务内**写入，避免"用户创建了但默认猫没发"或"账号建了但宝箱没初始化"等节点 2 §4.2 验收硬伤；任何一步失败 → 整体回滚 → 客户端拿到 1009 → SilentRelogin 兜底走重试链路（Story 5.4 落地）。

**AC1 — `internal/app/http/handler/auth_handler.go`：游客登录 handler**

新增 `server/internal/app/http/handler/auth_handler.go`，提供：

```go
// Package handler 已存在（含 ping / version handler）；本 story 加 auth_handler.go
package handler

// AuthHandler 是 /auth/* 子组的 handler 集合；本 story 仅提供 GuestLogin。
type AuthHandler struct {
    svc service.AuthService  // interface（service 层定义，便于单测 mock）
}

// NewAuthHandler 构造 AuthHandler。注入 AuthService（service 层）。
func NewAuthHandler(svc service.AuthService) *AuthHandler {
    return &AuthHandler{svc: svc}
}

// GuestLoginRequest 是 V1 §4.1 钦定请求体的 Go 端 mirror。
//
// 严格对齐 V1 §4.1 行 144-152 字段类型 / 必填 / 长度约束：
//   - guestUid: 1-128 字符（utf8.RuneCountInString，不是字节数；按 V1 §2.5 钦定）
//   - device.platform: enum "ios" / "android"（节点 2 仅 "ios"）
//   - device.appVersion: 1-32 字符
//   - device.deviceModel: 1-64 字符
//
// 任一字段缺失 / 长度超限 / platform 不在枚举 → 1002 参数错误（V1 §4.1 行 217）
type GuestLoginRequest struct {
    GuestUID string `json:"guestUid" binding:"required"`
    Device   GuestLoginDevice `json:"device" binding:"required"`
}

type GuestLoginDevice struct {
    Platform    string `json:"platform" binding:"required"`     // "ios" / "android"
    AppVersion  string `json:"appVersion" binding:"required"`
    DeviceModel string `json:"deviceModel" binding:"required"`
}

// GuestLogin 处理 POST /api/v1/auth/guest-login。
//
// 流程：
//  1. ShouldBindJSON 解析 + Gin binding:"required" 兜一层
//  2. **手动**校验长度（utf8.RuneCountInString）+ platform enum：Gin binding 不
//     做长度 / 枚举检查（详见 AC1 反模式段）
//  3. 调 svc.GuestLogin(ctx, GuestLoginInput{guestUid, platform, appVersion, deviceModel})
//  4. 成功 → response.Success(c, GuestLoginResponse{token, user, pet}, "ok")
//  5. 失败 → c.Error(err) + return（让 ErrorMappingMiddleware 写 envelope）
//
// **严格不**直接调 response.Error 写 1002 / 1009 envelope —— ADR-0006 钦定 "ErrorMapping 是
// 唯一 envelope 生产者"，所有错误一律 c.Error + return；middleware 兜底写 envelope。
//
// ctx 来源：c.Request.Context()（ADR-0007 §2.2 钦定，**不**用 *gin.Context 当 ctx）。
func (h *AuthHandler) GuestLogin(c *gin.Context) {
    var req GuestLoginRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        // BindJSON 失败 → 字段缺失 / 类型错；映射 1002
        _ = c.Error(apperror.Wrap(err, apperror.ErrInvalidParam, apperror.DefaultMessages[apperror.ErrInvalidParam]))
        return
    }

    // 长度校验（utf8.RuneCountInString，不是 len()）—— V1 §2.5 钦定按 Unicode 字符数
    if n := utf8.RuneCountInString(req.GuestUID); n < 1 || n > 128 {
        _ = c.Error(apperror.New(apperror.ErrInvalidParam, "guestUid 长度必须在 1-128 字符"))
        return
    }
    if n := utf8.RuneCountInString(req.Device.AppVersion); n < 1 || n > 32 {
        _ = c.Error(apperror.New(apperror.ErrInvalidParam, "appVersion 长度必须在 1-32 字符"))
        return
    }
    if n := utf8.RuneCountInString(req.Device.DeviceModel); n < 1 || n > 64 {
        _ = c.Error(apperror.New(apperror.ErrInvalidParam, "deviceModel 长度必须在 1-64 字符"))
        return
    }
    // platform 枚举校验：节点 2 仅 ios；android 是 schema 占位
    switch req.Device.Platform {
    case "ios", "android":
        // OK
    default:
        _ = c.Error(apperror.New(apperror.ErrInvalidParam, "platform 必须是 ios 或 android"))
        return
    }

    out, err := h.svc.GuestLogin(c.Request.Context(), service.GuestLoginInput{
        GuestUID:    req.GuestUID,
        Platform:    req.Device.Platform,
        AppVersion:  req.Device.AppVersion,
        DeviceModel: req.Device.DeviceModel,
    })
    if err != nil {
        _ = c.Error(err) // service 已 wrap apperror；handler 只透传
        return
    }

    response.Success(c, guestLoginResponseDTO(out), "ok")
}

// guestLoginResponseDTO 把 service 输出转成 V1 §4.1 钦定的 wire 格式。
//
// **关键**：BIGINT id 必须按 V1 §2.5 转字符串（避免 JS Number.MAX_SAFE_INTEGER 精度丢失）。
func guestLoginResponseDTO(out *service.GuestLoginOutput) gin.H {
    return gin.H{
        "token": out.Token,
        "user": gin.H{
            "id":             strconv.FormatUint(out.UserID, 10),
            "nickname":       out.Nickname,
            "avatarUrl":      out.AvatarURL,         // 首次创建为 ""（V1 §4.1 行 183）
            "hasBoundWechat": out.HasBoundWechat,    // 游客首次为 false（V1 §4.1 行 184）
        },
        "pet": gin.H{
            "id":      strconv.FormatUint(out.PetID, 10),
            "petType": out.PetType,                  // 节点 2 固定 1（V1 §4.1 行 186）
            "name":    out.PetName,                  // 首次创建 "默认小猫"（V1 §4.1 行 187）
        },
    }
}
```

**关键设计约束**：

- **handler 层只做参数校验 + 调 service + 返回 envelope**：不写业务逻辑（设计文档 §5.1 钦定）；不直接 import gorm / mysql 类型
- **`utf8.RuneCountInString` 而非 `len()`**：V1 §2.5 钦定按 Unicode 字符数计算长度（utf8mb4 1 字符可能 4 字节，`len()` 误判会拒绝合法多字节输入）
- **手动 enum 校验 platform**：Gin binding 标签不支持 enum（`binding:"oneof=ios android"` 写法对应 validator/v10 的 oneof，**可用**但本 story 选**显式 switch** —— 跨 reviewer 一目了然，不依赖第三方 validator 标签语法的隐式行为）；**注意**：android 节点 2 阶段是 schema 占位但 handler 不应硬拒（保持 future Android 端接入零改动）
- **`ShouldBindJSON` 而非 `BindJSON`**：BindJSON 失败时 Gin 会**自动写 400 + 文本 body**，绕过 ErrorMappingMiddleware；用 ShouldBindJSON 让我们自己处理（与 4.5 已建立模式一致）
- **错误一律 c.Error + return**：handler 不直接 response.Error 写 envelope（ADR-0006 单一生产者；与 4.5 中间件同模式）
- **id 转字符串用 strconv.FormatUint**：而非 `fmt.Sprintf("%d", ...)`（更快 + 不依赖 fmt reflect）
- **response.Success 第三参数 message 固定 "ok"**：V1 §2.4 envelope.message 在成功时为 "ok"（不是空串 / nil）
- **`hasBoundWechat` 字段**：游客首次创建固定 false；后续节点 2 阶段游客都是 false（绑定微信是 future epic）；本 story service 直接返 false 即可，**不**查 user_auth_bindings 看是否有 wechat 行（节点 2 永远没有）

**关键反模式**：

- ❌ **不**用 Gin binding 标签做长度校验（`binding:"min=1,max=128"`）—— 标签错误信息英文且 V1 §2.5 钦定按字符数；手动 utf8.RuneCountInString 更精准 + 错误信息可控
- ❌ **不**在 handler 直接调 `*gorm.DB`（违反 §5.1）
- ❌ **不**在 handler 内调 `signer.Sign(...)` —— 签 token 是 service 的事（service 知道 user.ID 后立即签）
- ❌ **不**用 `c.JSON(http.StatusBadRequest, ...)` 直接写 400 —— V1 §2.4 钦定业务码与 HTTP status 正交（1002 走 200 + envelope.code=1002，与 1001 同模式）
- ❌ **不**做"先调 svc.FindByGuestUID 看存不存在 / 再决定调 svc.Create" 拆分 —— 这个判断必须在 service 内做（handler 看不到 binding 表 / 看不到事务边界）；handler 永远调单一入口 `svc.GuestLogin`
- ❌ **不**接 `idempotencyKey` header —— V1 §4.1 钦定不需要（同 guestUid 自然幂等）
- ❌ **不**返回 `req.Device.Platform` / `req.Device.DeviceModel` 等任何 device 信息（V1 §4.1 response 只含 token / user / pet 三块；多返字段污染契约）

```go
// Package service 已存在（含 sample/）；本 story 加 auth_service.go
package service

// AuthService 是 auth handler 的依赖 interface（便于 handler 单测 mock）。
type AuthService interface {
    // GuestLogin: 复用或创建游客身份。
    //
    // 流程：
    //  1. authBindingRepo.FindByGuestUID(ctx, guestUid) 查 binding
    //  2. **命中**（err == nil + binding != nil）：
    //     - userRepo.FindByID(ctx, binding.UserID) 加载 user
    //     - petRepo.FindDefaultByUserID(ctx, user.ID) 加载默认 pet
    //     - signer.Sign(user.ID, 0) 签新 token
    //     - return GuestLoginOutput{...}
    //  3. **未命中**（err == ErrBindingNotFound）：开事务初始化 5 行（详见 AC2 事务段）
    //  4. **其他 err**（DB 异常）：apperror.Wrap(err, ErrServiceBusy)（兜底 1009）
    GuestLogin(ctx context.Context, in GuestLoginInput) (*GuestLoginOutput, error)
}

// GuestLoginInput / Output 是 service 层 DTO（不是 wire DTO，handler 转换）。
type GuestLoginInput struct {
    GuestUID    string
    Platform    string  // 节点 2 阶段不消费（future audit log 用）
    AppVersion  string
    DeviceModel string
}

type GuestLoginOutput struct {
    Token          string
    UserID         uint64
    Nickname       string
    AvatarURL      string
    HasBoundWechat bool
    PetID          uint64
    PetType        int
    PetName        string
}

// authServiceImpl 是 AuthService 的默认实装。
//
// 依赖（DI 注入）：
//   - txMgr: 事务管理器（4.2 落地）
//   - signer: JWT 签发（4.4 落地）
//   - userRepo / authBindingRepo / petRepo / stepAccountRepo / chestRepo: 5 个 mysql repo
//
// chestUnlockSec / petTypeDefault / petNameDefault / chestStatusCounting 等业务常量
// 在 service 内显式定义（service 层是业务规则的归属层）。
type authServiceImpl struct {
    txMgr            tx.Manager
    signer           *auth.Signer
    userRepo         repo.UserRepo
    authBindingRepo  repo.AuthBindingRepo
    petRepo          repo.PetRepo
    stepAccountRepo  repo.StepAccountRepo
    chestRepo        repo.ChestRepo
}

// NewAuthService 构造 AuthService。
func NewAuthService(
    txMgr tx.Manager,
    signer *auth.Signer,
    userRepo repo.UserRepo,
    authBindingRepo repo.AuthBindingRepo,
    petRepo repo.PetRepo,
    stepAccountRepo repo.StepAccountRepo,
    chestRepo repo.ChestRepo,
) AuthService {
    return &authServiceImpl{...}
}

// 业务常量（数据库设计 §5.6 / V1 §4.1 钦定）：
const (
    authTypeGuest          = 1                  // user_auth_bindings.auth_type=1 表示游客
    petTypeDefault         = 1                  // pets.pet_type=1 (cat)
    petCurrentStateRest    = 1                  // pets.current_state=1 (rest)
    petIsDefaultYes        = 1                  // pets.is_default=1
    petNameDefault         = "默认小猫"          // V1 §4.1 行 187
    userStatusActive       = 1                  // users.status=1
    chestStatusCounting    = 1                  // user_chests.status=1 (counting)
    chestOpenCostStepsDef  = 1000               // user_chests.open_cost_steps=1000（数据库设计 §5.6）
    chestUnlockDelay       = 10 * time.Minute   // 数据库设计 §6.7 + V1 §4.1 钦定首次创建 unlock_at = now + 10min
)

func (s *authServiceImpl) GuestLogin(ctx context.Context, in GuestLoginInput) (*GuestLoginOutput, error) {
    // (1) 查 binding（不开事务，普通连接池查询）
    binding, err := s.authBindingRepo.FindByGuestUID(ctx, in.GuestUID)
    if err != nil && !errors.Is(err, repo.ErrAuthBindingNotFound) {
        // DB 异常 → 1009
        return nil, apperror.Wrap(err, apperror.ErrServiceBusy, apperror.DefaultMessages[apperror.ErrServiceBusy])
    }

    // (2) 命中分支：复用
    if binding != nil {
        return s.reuseLogin(ctx, binding.UserID)
    }

    // (3) 未命中分支：开事务初始化 5 行
    return s.firstTimeLogin(ctx, in.GuestUID)
}

// reuseLogin 加载已有 user + 默认 pet → 签 token。
// 不开事务（只读 + 签 token，没有 multi-row 写入）。
func (s *authServiceImpl) reuseLogin(ctx context.Context, userID uint64) (*GuestLoginOutput, error) {
    user, err := s.userRepo.FindByID(ctx, userID)
    if err != nil {
        // 理论不应发生（binding 存在但 user 不存在 → 数据脏）；返 1009
        return nil, apperror.Wrap(err, apperror.ErrServiceBusy, apperror.DefaultMessages[apperror.ErrServiceBusy])
    }
    pet, err := s.petRepo.FindDefaultByUserID(ctx, userID)
    if err != nil {
        return nil, apperror.Wrap(err, apperror.ErrServiceBusy, apperror.DefaultMessages[apperror.ErrServiceBusy])
    }
    token, err := s.signer.Sign(user.ID, 0) // 0 = 用 default expire（4.4 钦定）
    if err != nil {
        return nil, apperror.Wrap(err, apperror.ErrServiceBusy, apperror.DefaultMessages[apperror.ErrServiceBusy])
    }
    return &GuestLoginOutput{
        Token:          token,
        UserID:         user.ID,
        Nickname:       user.Nickname,
        AvatarURL:      user.AvatarURL,
        HasBoundWechat: false, // 节点 2 阶段游客永远 false（V1 §4.1 行 184）
        PetID:          pet.ID,
        PetType:        int(pet.PetType),
        PetName:        pet.Name,
    }, nil
}

// firstTimeLogin: 5 行初始化事务（数据库设计 §8.1）。
//
// 事务内顺序（必须按此顺序，因为后 4 行依赖 user.ID）：
//   1. users 插入 → 拿到 user.ID（auto_increment）
//   2. user_auth_bindings 插入（type=1, identifier=guestUid, user_id=user.ID）
//   3. pets 插入（user_id=user.ID, pet_type=1, is_default=1, name="默认小猫"）
//   4. user_step_accounts 插入（user_id=user.ID, total_steps=0, available_steps=0, consumed_steps=0）
//   5. user_chests 插入（user_id=user.ID, status=1 counting, unlock_at=now+10min）
//
// 任一步失败 → tx.WithTx 自动 rollback → fn 返 error → service 返 1009
//
// nickname 计算：先插入 users（nickname 占位空串），拿到 user.ID 后**在事务内**
// 用 userRepo.UpdateNickname(txCtx, user.ID, "用户"+id) 写真实昵称。这是数据库
// 设计 §5.1 + V1 §4.1 行 182 钦定的"自动生成 nickname=用户{id}" 唯一可行实装路径
// （AUTO_INCREMENT id 必须先 INSERT 才知道，不能在 INSERT 前预填 nickname）。
//
// **唯一索引冲突处理**：user_auth_bindings.uk_auth_type_identifier(auth_type, auth_identifier)
// 在并发场景下可能冲突（两个并发请求带同一 guestUid，第一个创建 binding 后第二个 INSERT
// 被 MySQL 拒绝 ER_DUP_ENTRY 1062）。本 story 的处理：
//   - 把 ER_DUP_ENTRY 识别为 repo.ErrAuthBindingDuplicate（mysql repo 层做这件事）
//   - service 层捕获该 error → 视作"先入者刚刚创建" → 调 reuseLogin（递归一次）
//   - 这是 §AR1 资产事务原子原则的"幂等保证"实装：tx 内冲突 → tx 回滚 → service 把此错误
//     翻译为"binding 已存在" → 走复用分支
//   - 单测 case 覆盖（详见 AC8）
func (s *authServiceImpl) firstTimeLogin(ctx context.Context, guestUID string) (*GuestLoginOutput, error) {
    var (
        user *domain.User
        pet  *domain.Pet
    )

    err := s.txMgr.WithTx(ctx, func(txCtx context.Context) error {
        // (1) users 插入
        u := &domain.User{
            GuestUID:  guestUID,
            Nickname:  "", // 占位，下面 UpdateNickname 填
            AvatarURL: "",
            Status:    userStatusActive,
        }
        if err := s.userRepo.Create(txCtx, u); err != nil {
            return err
        }
        user = u

        // 写真实 nickname = "用户" + id（V1 §4.1 行 182）
        nickname := "用户" + strconv.FormatUint(u.ID, 10)
        if err := s.userRepo.UpdateNickname(txCtx, u.ID, nickname); err != nil {
            return err
        }
        u.Nickname = nickname

        // (2) user_auth_bindings 插入
        if err := s.authBindingRepo.Create(txCtx, &domain.AuthBinding{
            UserID:         u.ID,
            AuthType:       authTypeGuest,
            AuthIdentifier: guestUID,
        }); err != nil {
            return err
        }

        // (3) pets 插入（默认猫）
        p := &domain.Pet{
            UserID:       u.ID,
            PetType:      petTypeDefault,
            Name:         petNameDefault,
            CurrentState: petCurrentStateRest,
            IsDefault:    petIsDefaultYes,
        }
        if err := s.petRepo.Create(txCtx, p); err != nil {
            return err
        }
        pet = p

        // (4) user_step_accounts 插入（全 0）
        if err := s.stepAccountRepo.Create(txCtx, &domain.StepAccount{
            UserID:         u.ID,
            TotalSteps:     0,
            AvailableSteps: 0,
            ConsumedSteps:  0,
            Version:        0,
        }); err != nil {
            return err
        }

        // (5) user_chests 插入（counting + unlock_at = now + 10min）
        if err := s.chestRepo.Create(txCtx, &domain.UserChest{
            UserID:        u.ID,
            Status:        chestStatusCounting,
            UnlockAt:      time.Now().UTC().Add(chestUnlockDelay),
            OpenCostSteps: chestOpenCostStepsDef,
            Version:       0,
        }); err != nil {
            return err
        }

        return nil
    })

    if err != nil {
        // 检查是否是 binding 唯一冲突（并发场景）→ 走复用
        if errors.Is(err, repo.ErrAuthBindingDuplicate) {
            // 重新查 binding → 拿 user_id → reuseLogin
            binding, ferr := s.authBindingRepo.FindByGuestUID(ctx, guestUID)
            if ferr != nil || binding == nil {
                return nil, apperror.Wrap(err, apperror.ErrServiceBusy, apperror.DefaultMessages[apperror.ErrServiceBusy])
            }
            return s.reuseLogin(ctx, binding.UserID)
        }
        // 普通失败 → 1009（事务已 rollback）
        return nil, apperror.Wrap(err, apperror.ErrServiceBusy, apperror.DefaultMessages[apperror.ErrServiceBusy])
    }

    // 事务 commit 成功 → 签 token + 返回
    token, err := s.signer.Sign(user.ID, 0)
    if err != nil {
        // 极罕见：事务已 commit 但 sign 失败 —— 不应回滚（用户已创建）
        // 直接返 1009，client SilentRelogin 会重新登录拿到新 token
        return nil, apperror.Wrap(err, apperror.ErrServiceBusy, apperror.DefaultMessages[apperror.ErrServiceBusy])
    }

    return &GuestLoginOutput{
        Token:          token,
        UserID:         user.ID,
        Nickname:       user.Nickname,
        AvatarURL:      user.AvatarURL,
        HasBoundWechat: false,
        PetID:          pet.ID,
        PetType:        int(pet.PetType),
        PetName:        pet.Name,
    }, nil
}
```

**关键设计约束**：

- **service 是业务规则归属层**：`petTypeDefault=1` / `chestUnlockDelay=10*time.Minute` / `petNameDefault="默认小猫"` 等业务常量在 service 包定义，不在 handler / repo（设计文档 §5.2）
- **事务内顺序固定**：users → bindings → pets → step_accounts → chests（顺序由 user.ID 依赖决定）；调换顺序会让 §AR1 资产事务原子的"全有或全无"语义在 panic 边缘失败
- **nickname 两步写**：先 INSERT users（空串）拿 ID → UpdateNickname 写 `用户{id}`；这是 AUTO_INCREMENT 的语义约束（不能在 INSERT 前知道 id）
- **time.Now().UTC()**：V1 §2.5 钦定时间字段用 ISO 8601 UTC；MySQL DATETIME(3) 不带时区，存 UTC 让客户端按本地时区显示
- **唯一索引冲突 → 复用分支**：`user_auth_bindings.uk_auth_type_identifier` 冲突时回 reuseLogin（不直接返 1009）—— 节点 2 §AR1 资产事务原子 + 幂等保证
- **错误统一 wrap 为 1009**：service 层把 repo 错误一律 wrap 成 `ErrServiceBusy`（除了 ErrAuthBindingNotFound / ErrAuthBindingDuplicate 这种业务可识别的）；这与 V1 §4.1 行 219 钦定"1009 服务繁忙：数据库异常 / 事务回滚 / 内部 panic" 一致
- **HasBoundWechat 节点 2 永远 false**：游客首次 / 复用都是 false；future bind-wechat epic 落地时改成查 user_auth_bindings WHERE auth_type=2（wechat）
- **reuseLogin 不开事务**：只读 + 签 token，没有跨表写入；开事务反而 overhead

**关键反模式**：

- ❌ **不**在事务外做 5 行写入（违反数据库设计 §8.1 / §AR1）
- ❌ **不**在事务内 sign token（HMAC 是 CPU 计算，事务长度增加无意义；commit 后再 sign）
- ❌ **不**在 service 调 c.Get(...) / *gin.Context（service 不依赖 HTTP framework）
- ❌ **不**用 `string(rune('0' + id))` / `fmt.Sprintf("%d", id)` 转 id 字符串 → 用 strconv.FormatUint（更快 + 不依赖 fmt reflect）
- ❌ **不**在 firstTimeLogin "失败回滚 → 用户首次登录 → 客户端会重试" 路径加 sleep / retry 包装：service 直接返 1009，retry 由 iOS Story 5.4 SilentRelogin 处理（端职责分工）
- ❌ **不**预先 SELECT user_auth_bindings WHERE user_id=? 看是否绑了 wechat —— 节点 2 永远没有；多查一次浪费 RTT
- ❌ **不**捕获 `context.Canceled` / `context.DeadlineExceeded` 把它们 wrap 成 1009 —— ADR-0007 §6 钦定 ctx err 由 ErrorMappingMiddleware 兜底（apperror.Wrap 已让 errors.Is 穿透）
- ❌ **不**在事务里做 `time.Sleep` / 大量 CPU 计算（拉长事务持续时间，提高死锁概率）

```go
// user_repo.go
package mysql

import (
    "context"
    "errors"

    "github.com/go-sql-driver/mysql"
    "gorm.io/gorm"

    "github.com/huing/cat/server/internal/repo/tx"
)

type UserRepo interface {
    Create(ctx context.Context, u *User) error                              // 写
    UpdateNickname(ctx context.Context, userID uint64, nickname string) error // 写
    FindByID(ctx context.Context, id uint64) (*User, error)                 // 读
}

type userRepo struct {
    db *gorm.DB
}

func NewUserRepo(db *gorm.DB) UserRepo {
    return &userRepo{db: db}
}

// Create 用 tx.FromContext 拿 db handle（事务内用 tx；事务外用 fallback）。
// INSERT 后 GORM 自动回填 u.ID（AUTO_INCREMENT）。
//
// **必须** .WithContext(ctx) —— ADR-0007 §2.3 repo 必须 WithContext，否则 ctx cancel 不生效。
func (r *userRepo) Create(ctx context.Context, u *User) error {
    db := tx.FromContext(ctx, r.db)
    return db.WithContext(ctx).Create(u).Error
}

// UpdateNickname 用 GORM Updates 仅更新 nickname 字段（避免 zero value 覆盖）。
func (r *userRepo) UpdateNickname(ctx context.Context, userID uint64, nickname string) error {
    db := tx.FromContext(ctx, r.db)
    return db.WithContext(ctx).
        Model(&User{}).
        Where("id = ?", userID).
        Update("nickname", nickname).Error
}

// FindByID 查单行；NotFound → ErrUserNotFound 哨兵；其他 DB 异常透传。
func (r *userRepo) FindByID(ctx context.Context, id uint64) (*User, error) {
    db := tx.FromContext(ctx, r.db)
    var u User
    err := db.WithContext(ctx).Where("id = ?", id).First(&u).Error
    if err != nil {
        if errors.Is(err, gorm.ErrRecordNotFound) {
            return nil, ErrUserNotFound
        }
        return nil, err
    }
    return &u, nil
}
```

```go
// auth_binding_repo.go（关键差异：Create 必须识别 ER_DUP_ENTRY → ErrAuthBindingDuplicate）
package mysql

type AuthBindingRepo interface {
    Create(ctx context.Context, b *AuthBinding) error
    FindByGuestUID(ctx context.Context, guestUid string) (*AuthBinding, error) // 不存在返 nil + ErrAuthBindingNotFound
}

type authBindingRepo struct {
    db *gorm.DB
}

func NewAuthBindingRepo(db *gorm.DB) AuthBindingRepo {
    return &authBindingRepo{db: db}
}

func (r *authBindingRepo) Create(ctx context.Context, b *AuthBinding) error {
    db := tx.FromContext(ctx, r.db)
    err := db.WithContext(ctx).Create(b).Error
    if err != nil {
        // 识别 MySQL ER_DUP_ENTRY 1062
        var mysqlErr *mysql.MySQLError
        if errors.As(err, &mysqlErr) && mysqlErr.Number == 1062 {
            return ErrAuthBindingDuplicate
        }
        return err
    }
    return nil
}

func (r *authBindingRepo) FindByGuestUID(ctx context.Context, guestUid string) (*AuthBinding, error) {
    db := tx.FromContext(ctx, r.db)
    var b AuthBinding
    err := db.WithContext(ctx).
        Where("auth_type = ? AND auth_identifier = ?", authTypeGuest, guestUid).
        First(&b).Error
    if err != nil {
        if errors.Is(err, gorm.ErrRecordNotFound) {
            return nil, ErrAuthBindingNotFound
        }
        return nil, err
    }
    return &b, nil
}

const authTypeGuest = 1 // 与 service 同源，但 repo 层独立定义（避免循环 import）
```

**关键设计约束**：

- **repo 层是数据访问的唯一抽象**：service 不直接 import gorm（仅 import repo interface + repo 哨兵 error）；repo 内部封装 GORM API + 错误识别（ER_DUP_ENTRY / ErrRecordNotFound）
- **`tx.FromContext(ctx, r.db)` 模式**：ctx 里有 tx → 用 tx；否则用 fallback。事务内调 `Create(txCtx, ...)` 走 tx；事务外调 `FindByID(ctx, ...)` 走 db pool。这是 4.2 已建立的范式
- **`.WithContext(ctx)` 必须**：ADR-0007 §2.3 钦定 repo 必须用 WithContext，否则 ctx cancel 不生效
- **域模型 = GORM struct**：MVP 选择不做 domain 与 ORM 分离（4.2 sample/ 已建立先例）；future 业务增长再拆
- **哨兵 error**：repo 层暴露两类哨兵（ErrXxxNotFound / ErrXxxDuplicate），service 用 `errors.Is` 区分；其他 DB error 原样透传
- **ER_DUP_ENTRY 识别**：用 `errors.As(err, &mysqlErr)` + `mysqlErr.Number == 1062`（go-sql-driver/mysql 已 export 该 struct + 错误码常量；不要硬编码字符串匹配）
- **不在 repo 写业务规则**：repo 只做 CRUD；如 `petTypeDefault=1` 这种业务常量在 service 层定义（除 `authTypeGuest=1` 因为 binding repo 自己 query 时需要 → 在 repo 内独立常量）

**关键反模式**：

- ❌ **不**在 repo 调 `txMgr.WithTx(...)`（事务边界归 service 控制；§5.2）
- ❌ **不**在 repo 返回业务码 / apperror（repo 只产原生 error + 哨兵；§AppError 三层映射 ADR-0006）
- ❌ **不**省略 `.WithContext(ctx)`（违反 ADR-0007 §2.3）
- ❌ **不**用 `db.Create(&u).Error` 而漏 ctx（同上）
- ❌ **不**用字符串匹配识别 MySQL 错误（"Error 1062" / "Duplicate entry" 不可靠 + 国际化版本不同）；用 `*mysql.MySQLError` + Number

```go
import (
    "github.com/huing/cat/server/internal/app/http/handler"
    "github.com/huing/cat/server/internal/app/http/middleware"
    "github.com/huing/cat/server/internal/repo/mysql"
    "github.com/huing/cat/server/internal/service"
)

func NewRouter(deps Deps) *gin.Engine {
    r := gin.New()
    _ = r.SetTrustedProxies(nil)
    r.Use(middleware.RequestID())
    r.Use(middleware.Logging())
    r.Use(middleware.ErrorMappingMiddleware())
    r.Use(middleware.Recovery())

    r.GET("/ping", handler.PingHandler)
    r.GET("/version", handler.VersionHandler)
    r.GET("/metrics", gin.WrapH(metrics.Handler()))

    devtools.Register(r)

    // ★ 4.6 新增：业务路由组
    api := r.Group("/api/v1")

    // 4.6 wire：仅当 deps 完整时才挂业务路由（测试场景 Deps{} 零值不挂，避免 nil pointer）
    if deps.GormDB != nil && deps.TxMgr != nil && deps.Signer != nil {
        // 5 个 repo
        userRepo := mysql.NewUserRepo(deps.GormDB)
        authBindingRepo := mysql.NewAuthBindingRepo(deps.GormDB)
        petRepo := mysql.NewPetRepo(deps.GormDB)
        stepAccountRepo := mysql.NewStepAccountRepo(deps.GormDB)
        chestRepo := mysql.NewChestRepo(deps.GormDB)

        // service
        authSvc := service.NewAuthService(deps.TxMgr, deps.Signer, userRepo, authBindingRepo, petRepo, stepAccountRepo, chestRepo)

        // handler
        authHandler := handler.NewAuthHandler(authSvc)

        // /auth 子组：rate_limit by IP（V1 §4.1 行 218 钦定），**不**挂 auth
        // RateLimit 工厂在 NewRouter 调用期校验 cfg（PerKeyPerMin <= 0 → panic）；
        // 与 4.5 fail-fast 模式一致。
        authGroup := api.Group("/auth", middleware.RateLimit(deps.RateLimitCfg, middleware.RateLimitByIP))
        authGroup.POST("/guest-login", authHandler.GuestLogin)

        // 已认证子组：4.8 落地时填 GET /home；本 story 留空
        // authedGroup := api.Group("",
        //     middleware.Auth(deps.Signer),
        //     middleware.RateLimit(deps.RateLimitCfg, middleware.RateLimitByUserID),
        // )
        // authedGroup.GET("/home", homeHandler.Home)  // 4.8 加
    }

    return r
}
```

**关键设计约束**：

- **deps 完整性 if-guard**：`deps.GormDB != nil && deps.TxMgr != nil && deps.Signer != nil` 才挂业务路由 —— 单元测试 Deps{} 零值场景（`server_test.go` / `router_test.go`）保持 zero-deps 路径，**不** nil panic
- **`/auth` 子组 RateLimitByIP**：与 4.5 godoc 钦定形态一致；V1 §4.1 行 218 钦定 IP 维度
- **wire 在 NewRouter 内**：而非 main.go —— router 工厂是依赖组装的归属层；main.go 只负责构造 deps（fail-fast 校验）+ 透传给 NewRouter
- **不引入 wire / fx 框架**：显式 DI（与 4.2 / 4.4 / 4.5 同模式）；5 个 repo + 1 service + 1 handler 共 7 行手动 wire，可读 + 可维护

**关键反模式**：

- ❌ **不**在 main.go 直接 wire 5 个 repo（会让 main.go 膨胀；router 是依赖组装层）
- ❌ **不**给 deps 完整性兜底（如 `if deps.Signer == nil { deps.Signer = mockSigner }`）—— 违反 fail-fast；要么完整要么不挂业务路由
- ❌ **不**把 service / repo 注册当 init() 副作用（隐式注册导致测试场景污染）
- ❌ **不**让 /auth/guest-login 走全局中间件（之前 4.5 godoc 钦定的形态严格区分 /auth 子组的 IP rate_limit vs 已认证子组的 userID rate_limit）

## Dev Notes

### 关键设计原则

1. **§AR1 资产事务原子严格落地**：5 行写入必须同一事务（数据库设计 §8.1 + epics.md AR1 钦定）；任何一步失败 → tx.WithTx 自动 rollback → service 包装为 1009；client 侧由 Story 5.4 SilentRelogin 兜底重试。**禁止**任何"先创建 user，后台异步补 chest"的拆分实装。
2. **同 guestUid 自然幂等**：靠 `user_auth_bindings.uk_auth_type_identifier` 唯一索引；ER_DUP_ENTRY 1062 时 service 回退到 reuseLogin（先入者刚创建，本次走复用）。**不**接 Redis idempotencyKey（V1 §4.1 行 141 + 节点 2 §AR3 钦定）。
3. **nickname 两步写**：先 INSERT users（nickname 占位空串）→ 拿到 auto_increment id → 第二步 UPDATE nickname = `用户{id}`。AUTO_INCREMENT 语义约束所致；不存在"INSERT 时一步到位"的实装路径（除非用 trigger，违反 §5.1 "service 是业务规则归属层"）。
4. **time.Now().UTC() + DATETIME(3)**：V1 §2.5 钦定时间字段 ISO 8601 UTC；MySQL DATETIME(3) 不带时区，存 UTC 让客户端按本地时区显示；与 4.3 migrations 已建立的 `created_at DATETIME(3)` 字段语义一致。
5. **service / repo 分层严格**：service 持业务常量 + 事务边界 + 调多 repo 编排；repo 只做单表 CRUD + 错误识别（ER_DUP_ENTRY → 哨兵）；handler 只做参数校验 + 调 service + 返 envelope。三层不串味（ADR-0006 + 设计文档 §5）。
6. **fail-fast over fallback**：bootstrap deps 完整性 if-guard（`deps.GormDB != nil && ...`）—— 单测 Deps{} 零值不挂业务路由，避免 nil pointer；生产路径 main.go 已 fail-fast（4.2 / 4.4 / 4.5 build），所以 deps 完整性在 NewRouter 是 invariant。
7. **不依赖 ctx 取消业务流程**：repo 调 `.WithContext(ctx)` 让 GORM 感知 client 断开；service 不主动 select ctx.Done()（事务内 ctx cancel 自然 propagate 到 SQL → 触发 rollback）；handler 拿 `c.Request.Context()` 透传（ADR-0007 §2.2）。

### 关键决策点（实装时注意）

1. **GORM auto_increment 回填**：`db.Create(&user)` 后 user.ID 自动被 GORM 回填（前提：struct 有 `gorm:"primaryKey;autoIncrement"` tag）。**关键**：必须**先**用 `&user`（指针）调 Create，不能传值类型 `db.Create(user)` —— 后者 ID 不回填。

2. **MySQL ER_DUP_ENTRY 1062 识别**：

   ```go
   import "github.com/go-sql-driver/mysql"

   var mysqlErr *mysql.MySQLError
   if errors.As(err, &mysqlErr) && mysqlErr.Number == 1062 {
       return ErrAuthBindingDuplicate
   }
   ```

   `mysql.MySQLError.Number` 是 `uint16`；常量 `1062` 是 ER_DUP_ENTRY；`go-sql-driver/mysql` 已在 go.mod indirect dep 里（4.2 引入），本 story 提升为 direct dep（go.mod direct 段）。

3. **GORM Updates vs Update vs Save**：

   - `Update("nickname", v)` — 更新单字段（推荐用于 UpdateNickname）
   - `Updates(map[string]any{...})` — 多字段；忽略 zero value（除非用 Updates(struct)）
   - `Save(&u)` — 全字段 UPDATE（包括 zero value 覆盖；**禁用** —— 会把 created_at 也覆盖）
   - **本 story UpdateNickname 用 `Update` 单字段**

4. **GORM `.WithContext(ctx)` 必须**：

   ```go
   db.WithContext(ctx).Create(u)        // ✅
   db.Create(u)                          // ❌ ctx cancel 不感知，违反 ADR-0007 §2.3
   ```

   即使 `tx.FromContext(ctx, r.db)` 已经从 ctx 拿 tx，**仍**要 `.WithContext(ctx)` —— FromContext 只取 db handle，WithContext 才让 GORM 感知 cancel 信号。

5. **time.Now().UTC()** vs **time.Now()**：

   ```go
   chest.UnlockAt = time.Now().UTC().Add(10 * time.Minute)  // ✅
   chest.UnlockAt = time.Now().Add(10 * time.Minute)        // ❌ 带本地时区
   ```

   MySQL DATETIME(3) 不带时区，按"将 Go time.Time 字面量字段值原样存"行为；如果 `time.Now()` 是 +08:00（北京）会被存为北京时间字面量，但 V1 §2.5 钦定 ISO 8601 UTC 下发 → 客户端按 UTC 解析 → 结果是"北京时间字面量当 UTC 解析" = 实际多 8 小时偏差。`.UTC()` 强制转 UTC 时区视图，存进 DATETIME(3) 是 UTC 字面量，下发 ISO 8601 UTC 后客户端解析无偏差。

6. **utf8.RuneCountInString vs len()**：V1 §2.5 钦定按 Unicode 字符数：

   ```go
   utf8.RuneCountInString("中国")  // = 2
   len("中国")                     // = 6（utf8mb4 每字符 3 字节）
   ```

   guestUid 长度 1-128 字符是按 RuneCount，不是字节数。

7. **strconv.FormatUint(id, 10)** vs **fmt.Sprintf("%d", id)**：strconv 更快 + 不依赖 fmt reflect；handler 转 BIGINT id 用 strconv（与 V1 §2.5 钦定的"内部 uint64 / wire string" 划分一致）。

8. **stub repo / stub txMgr 命名**：

   ```go
   type stubUserRepo struct {
       createFn func(ctx context.Context, u *mysql.User) error
       findByIDFn func(ctx context.Context, id uint64) (*mysql.User, error)
       // ...
   }

   func (s *stubUserRepo) Create(ctx context.Context, u *mysql.User) error {
       return s.createFn(ctx, u)
   }
   // ...
   ```

   每个 stub 只实装 service / handler 用到的 method（不需要全部 method），通过 interface 契约保证编译期检查。

9. **stubTxMgr.WithTx 直接调 fn**：


**关键决策点**：
1. **事务编排顺序**：users INSERT → users UPDATE nickname → bindings INSERT → pets INSERT → step_accounts INSERT → chests INSERT。后 5 步依赖 user.ID（AUTO_INCREMENT 必须先 INSERT 才知道）；nickname 两步写是 `用户{id}` 的唯一可行实装路径（不用 trigger 是为遵循设计文档 §5.2 业务规则归属 service 层）。

4. **token 签发集成**：reuseLogin 与 firstTimeLogin 都在**事务 commit 之后**调 `signer.Sign(user.ID, 0)`（HMAC 是 CPU 计算，不应拉长事务持续时间）；Sign 失败极罕见，依然返 1009 让 client SilentRelogin 重试（不主动 rollback —— 用户已落库）。
