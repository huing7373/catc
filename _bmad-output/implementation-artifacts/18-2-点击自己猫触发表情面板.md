### 决策点：sheet vs custom overlay

epics.md 行 2666 钦定"用 SwiftUI sheet 或自定义 overlay" —— 二选一允许。**本 story 选 SwiftUI `.sheet(isPresented:)` 模式**：

1. **样板成熟**：HomeView `JoinRoomModal` sheet 已落地（Story 37.12）+ 5 轮 review 沉淀踩坑（详见 HomeView.swift line 102-126 `onDismiss` 闭包注释 + lesson `2026-04-30-sheet-on-dismiss-button-vs-swipe-driven.md`）→ 本 story 直接抄
2. **ADR-0009 §3.3 sheet 白名单兼容**：白名单第 2 行 "JoinRoomModal `.sheet` (bottom sheet) HomeView 内" 是先例；本 story 同属"Tab 内部次级 sheet"语义扩展（**不**破 ADR；如未来 ADR 形式收窄白名单到 enum，本 story 走"扩展白名单一项 EmojiPanel"路径补 ADR；本 story 不开新 ADR，仅在 Dev Notes 标注"如未来收紧 ADR-0009 §3.3 白名单需补本场景"）
3. **生命周期受控**：`.sheet(isPresented: $vm.showEmojiPanel)` 双向绑定让 sheet open / close 状态机走 ViewModel @Published 单一 owner，避免 SwiftUI `@State` 双写 → 与 ADR-0010 §3.2 表格行"Sheet 是否打开 → ViewModel @Published var showXModal"严格对齐
4. **自定义 overlay 缺点**：(a) 需要自己写 transition / dismiss 手势 / safe area 适配 → 增加 50+ 行重复样板代码；(b) 没有自动 swipe-down 关闭；(c) 与 EmojiPanelView 内部 `.task { await viewModel.load() }` 生命周期不天然兼容（overlay 隐藏时 view tree 不一定 unmount，可能让二次显示 `.task` 不重跑 → 但因 18.1 LoadEmojisUseCase actor 缓存共享，二次访问不会重发 GET /emojis，实际无功能影响；仍是 sheet 路径更稳）

### 决策点：currentUserId 注入路径

epics.md 行 2666 未直接钦定 self userId 派生路径。**本 story 选 `RoomViewModel.@Published var currentUserId: String?` 由 `RealRoomViewModel` 订阅 `appState.$currentUser` 派生**：

1. **唯一权威源**：AppState.currentUser 是 ADR-0010 §3.1 钦定的"server 决定的事实"single source of truth；任何派生（含本 story self vs other 判定）都应订阅它，**禁止**让 RoomScaffoldView 直接读 `@EnvironmentObject var appState: AppState` —— 理由：(a) RoomScaffoldView 已经 `@ObservedObject var state: RoomViewModel` 走 ViewModel 模式，再加 `@EnvironmentObject` 就两源；(b) MockRoomViewModel 路径下 RoomScaffoldView Preview / UITest 不一定挂 AppState environment；(c) Story 37.4 ADR-0010 §3.1 钦定"View 只通过 ViewModel 间接访问 AppState"
2. **Mock 注入友好**：MockRoomViewModel 增 `currentUserId` 参数，单元测试 `MockRoomViewModel(currentUserId: "u1", members: [.u1 自己, .u2 他人])` 一句话注入；不需要构造完整 AppState（与 MockHomeViewModel 不持 AppState 同精神）
3. **reset 路径自动同步**：appState.currentUser → nil 时 currentUserId 自动 → nil（防 reset 后老 self 残留 → RoomScaffoldView "自己位"判定卡在已注销的 userId → 一旦 user 重 hydrate 拿到新 id 还会出现"两个自己"短暂残留）
4. **不在 RoomScaffoldView 内 hardcode**：禁止用 "members[0].id 当 self" 偷懒（房主可能不是自己；多人房间下 host 可能是别的成员）—— 必须走 currentUserId 明确判定

### 决策点：member.id == currentUserId 的视图层防御 vs ViewModel 层防御

epics.md 行 2670 钦定"别人的猫位**不可点击**触发表情面板（防误操作）"。**两层防御**：

- **View 层**（本 story 主路径）：`memberRow` 内对 `member.id == state.currentUserId` 用 Button 包裹 PetSpriteView；非自己行直接 PetSpriteView 不包 Button → 物理上"别人位无 Button → UITest `app.buttons["roomMember_1_petSprite"]` 不存在 → 不可能触发 onOwnPetTap"
- **ViewModel 层**（本 story **不**实装额外校验）：`onOwnPetTap()` 内部不再校验"调用方是不是自己"——理由：(a) 调用方只可能是 View 层 Button（View 层已限定只在自己行渲染 Button）；(b) 如未来路径变（如机器人自动调 vm.onOwnPetTap）属于 caller 误用，不在本 story 防御边界；(c) ViewModel 层无 self userId 可校验入参（onOwnPetTap 无入参）——保持 abstract method 签名最简

## Dev Notes

### 关键架构约束

1. **唯一 owner 原则（ADR-0010 §3.2）**：
   - `showEmojiPanel` 唯一 owner = `RoomViewModel` @Published；RoomScaffoldView 通过 `$state.showEmojiPanel` 双向绑定，**不**用 SwiftUI `@State` 重复持有
   - `currentUserId` 唯一权威源 = `RealRoomViewModel` 从 `appState.$currentUser.map { $0?.id }` 派生；RoomScaffoldView 通过 `state.currentUserId` 间接访问，**不**用 `@EnvironmentObject var appState: AppState` 直接读

2. **abstract method 模式（既有 RoomViewModel 模式延续）**：
   - 基类 `onOwnPetTap()` 走 `fatalError` 占位让子类强制 override；与 `onLeaveTap()` / `onCopyTap()` 同模式
   - MockRoomViewModel override：记录 invocation + 状态切换
   - RealRoomViewModel override：纯本地状态切换（不调 server / WS）

3. **sheet 模式 vs custom overlay（ADR-0009 §3.3 sheet 白名单语义扩展）**：
   - 选 `.sheet(isPresented:)` 路径；样板抄 HomeView JoinRoomModal（Story 37.12 已稳定 5 轮 review）
   - **不**开新 ADR；如未来 ADR-0009 §3.3 收紧白名单为 enum，再补本场景到白名单

4. **闭包注入 factory（避免 RoomScaffoldView 持 AppContainer 依赖）**：
   - `emojiPanelViewModelFactory: () -> EmojiPanelViewModel` 由 caller（RootView）注入
   - sheet 内 `EmojiPanelView(viewModel: emojiPanelViewModelFactory(), ...)` 每次 sheet 弹出 new 一个 vm（@StateObject 在 sheet 内持有直到 dismiss）；vm 内 useCase 是 container 单例 → cache 跨 sheet 共享（与 18.1 缓存契约一致）

5. **`.buttonStyle(.plain)` 必须挂**：
   - 默认 `DefaultButtonStyle` 给 PetSpriteView SF Symbol 染蓝 → 视觉破坏
   - 与任何 Button-wraps-Image 路径同 lesson（不需要新 lesson，遵循 SwiftUI 已知行为）
