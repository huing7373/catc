# Story 8.3: 运动状态机映射（CMMotion → rest/walk/run）

Status: review

> **fix-review r1 addendum（2026-05-04，codex P2）**：原 AC2 / AC3 钦定 "confidence == .low → 保持 previous 防抖" 在 fix-review r1 被认定为 over-spec：
> - **下游冲突**：低置信度 stationary 时 mapper 返 previous → 8.4 HomeViewModel.petState / 8.5 /steps/sync motionState 卡在 stale `.walk` / `.run`，用户停下却不显示 .rest.

> - **保留 previous 参数**：API 签名保留接受 `previous: MotionState?` 但实装内不消费，给未来 ViewModel 层若复用 mapper 签名做 hysteresis 留接口.

## Story

As an iOS 开发,
I want 一个 `MotionStateMapper`（pure function）把原始 `CMMotionActivity` 转换成业务三态 `MotionState { rest, walk, run }`,
so that 上游业务层（Story 8.4 `HomeViewModel.petState` 订阅 / Story 8.5 同步触发器拼 `motionState` 字段 / Story 9.1 跨端 e2e 验证场景 4）不直接处理底层 CoreMotion 类型，状态映射规则集中在一处可测，并满足 docs/宠物互动App_iOS客户端工程结构与模块职责设计.md §10.2 钦定的"`stationary → rest` / `walking → walk` / `running → run`" 映射契约。

### AC1 — `MotionState` enum 定义（新文件 `iphone/PetApp/Core/Motion/MotionState.swift`）

新建 `iphone/PetApp/Core/Motion/MotionState.swift`，按 docs/宠物互动App_iOS客户端工程结构与模块职责设计.md §10.2 钦定 + epics.md AC 行 1502 钦定：

```swift
// MotionState.swift
// Story 8.3 AC1: 业务三态运动状态枚举（System Adapter 层 → Domain 层之间的统一类型基础）.
//
// 设计基线（详见 story 8-3-运动状态机映射.md AC1 段）:
// - 三态闭集：.rest / .walk / .run（与 docs §10.2 钦定的"rest / walk / run"映射目标对齐）
// - 不含 .unknown / .cycling / .automotive 等 system framework 概念——这些由 MotionStateMapper.map(_:)
//   按规则归并到 .rest（详见 AC2）；ViewModel / UI 层只感知三态闭集
// - String rawValue 直接用作 V1 §6.1 /steps/sync 的 motionState 字段值（小写英文，与契约对齐）
//   - "rest" / "walk" / "run"（不大写、不带前缀，避免 8.5 拼请求体时再做 lowercased() 转换）
// - Codable + Equatable + Sendable + CaseIterable（CaseIterable 给 8.4 PetSpriteView 测试遍历状态用）
// - 不引 import CoreMotion（pure 业务类型；与协议层 MotionProvider.swift 区分——后者必 import CM）

import Foundation

/// 业务三态运动状态枚举.
/// - rest: 静止 / 坐下 / 系统未识别活动 / cycling / automotive / 低置信度兜底（详见 MotionStateMapper.map(_:)）
/// - walk: 行走（含慢走）
/// - run: 跑步（含快跑）
public enum MotionState: String, Codable, Equatable, Sendable, CaseIterable {
    case rest = "rest"
    case walk = "walk"
    case run = "run"
}
```

**关键设计决策（必须严格遵守）**：

- **String rawValue 用小写英文**：V1 §6.1 `/steps/sync` 钦定的 `motionState` 字段值是 `"rest"` / `"walk"` / `"run"`（参考 docs/宠物互动App_V1接口设计.md §6.1）；本 enum 直接 `String` raw 让 Story 8.5 `SyncStepsUseCase` 拼 JSON body 时 `motionState: petState.rawValue` 一行搞定，避免 8.5 再写 enum→String 翻译表。**禁止**用 `.rest = "REST"` / `.rest = "Rest"` 等大小写变体。
- **三态闭集 + 不引 .unknown / .cycling / .automotive 等系统态**：这是 epics.md AC 行 1502-1507 钦定的"业务三态"——业务层只关心 rest / walk / run；其他系统态（cycling / automotive / unknown）由 mapper 归并到 .rest（docs §10.2 "坐下按静止处理"同精神扩展到所有非 walk/run 状态）。**禁止**把 .cycling 加进 enum；如未来产品要分骑行 / 自驾，Story 9.x 再扩。
- **`Codable + Sendable`**：Codable 给 Story 8.5 的 `/steps/sync` 请求体 JSON 编码用；Sendable 给 Swift 6 strict concurrency 跨 actor 传递用（Story 8.4 HomeViewModel `@Published petState` 在 main actor，订阅来源在 mock / impl 的 OperationQueue.main 触发，自然走 main，但 Sendable 是合规标记）。
- **`CaseIterable`**：让 8.4 PetSpriteView 测试可 `MotionState.allCases.forEach { ... }` 遍历三态验证 sprite 切换；给 mapper 测试也方便（**不**强求本 story 用 allCases，但加上零成本）。
- **`Equatable` 必加**：mapper 测试 `XCTAssertEqual(mapped, .walk)` 必须的；enum + String raw 自动 Equatable，但显式标 `Equatable` 给协议约束 explicit。
- **不放在 `MotionStateMapper.swift` 同文件**：架构 §4 钦定 `Core/Motion/MotionState.swift` 单独文件（和 `MotionStateProvider.swift` / `MotionPermissionManager.swift` 同级；本 story 实际命名 `MotionState.swift` + `MotionStateMapper.swift`，与架构 §4 命名收口逻辑对齐 — 见"Project Structure Notes"）。**禁止**把 enum 内联到 mapper 文件以"少一个文件"为由。

### AC2 — `MotionStateMapper.map(_:)` pure function 实装（新文件 `iphone/PetApp/Core/Motion/MotionStateMapper.swift`）

新建 `iphone/PetApp/Core/Motion/MotionStateMapper.swift`：

```swift
// MotionStateMapper.swift
// Story 8.3 AC2: CoreMotion CMMotionActivity → 业务三态 MotionState 的映射规则集中点.
//
// 设计基线（详见 story 8-3-运动状态机映射.md AC2 段）:
// - pure function（struct + static func），无状态外部依赖
//   - **例外**：confidence 低置信度防抖路径需要"上一次状态"——通过显式参数 `previous: MotionState?`
//     传入由 caller（Story 8.4 HomeViewModel）持有；mapper 自身仍是 pure（同输入同输出）
// - 优先级规则：running > walking > stationary > 其他
//   - epics.md AC 行 1514 钦定"多个 flag 同时 true（如 walking + stationary）→ 优先级 run > walk > rest"
//   - cycling / automotive / unknown / 其他全部归 .rest（docs §10.2 "坐下按静止处理" 同精神扩展）
// - confidence 低置信度策略：< .low（即 .low 自身不算低；只有未来 SDK 可能添加的 .min 等才算）
//   - epics.md AC 行 1515 钦定"confidence < .low → 保持上一次状态（防抖）"
//   - **但 CMMotionActivityConfidence 当前枚举只有 .low / .medium / .high 三档**（iOS 17+ 公开 API）；
//     "< .low" 在当前 SDK 实际是空集——为了对齐 AC 钦定语义且 forward-compat，本实装策略：
//     `if activity.confidence == .low { return previous ?? .rest }`（最低档视作"低置信度"防抖入口）
//     理由 1）docs §10.2 没明确 confidence 阈值；2）epics.md AC 钦定 "< .low 保持上一次"是 forward-looking
//     描述（防 Apple 未来扩 .veryLow / .min 档）；3）当前 .low 是数值最小档，把 .low 视作"低置信度"
//     起到"防抖"实际效果（实测模拟器 .low 占比 30%+，频繁切换会造成 PetSpriteView 闪烁）。
// - previous 参数 default = nil：caller 不传时（首次调用 / 无历史状态）低置信度归 .rest（兜底中性态）

import Foundation
import CoreMotion  // 输入是 CMMotionActivity；本文件唯一引 CoreMotion 的 pure 业务文件

/// CoreMotion CMMotionActivity → 业务三态 MotionState 的纯函数映射器.
/// pure function（struct + static func）；无状态、无副作用、无外部依赖；同输入同输出.
///
/// 规则（按优先级从高到低）:
///   1. confidence 低置信度（== .low）→ 返回 `previous ?? .rest`（防抖：保持上一次状态；首次调用兜底 .rest）
///   2. running == true → .run
///   3. walking == true → .walk
///   4. stationary == true → .rest
///   5. cycling / automotive / unknown / 其他（含全 false 兜底）→ .rest
///
/// 优先级与 epics.md §Story 8.3 AC 行 1514 钦定"多个 flag 同时为 true → run > walk > rest"严格对齐.
/// docs/宠物互动App_iOS客户端工程结构与模块职责设计.md §10.2 钦定的"stationary → rest" / "walking → walk" /
/// "running → run" 是规则 2-4 的来源；规则 5 是 "坐下按静止处理"语义扩展（cycling / automotive 等也按"非 walk/run 即 rest"归并）.
public enum MotionStateMapper {
    /// 把 CMMotionActivity 映射到业务三态 MotionState.
    /// - Parameters:
    ///   - activity: 系统识别到的 motion activity 实例（含 stationary/walking/running/cycling/automotive/unknown 多个 Bool flag + confidence + startDate）.
    ///   - previous: caller 持有的上一次映射结果（可选）；用于 confidence == .low 时保持状态防抖.
    ///                首次调用 / 无历史状态时传 nil（兜底 .rest）.
    /// - Returns: 业务三态 MotionState.
    public static func map(_ activity: CMMotionActivity, previous: MotionState? = nil) -> MotionState {
        // 规则 1：低置信度防抖优先于 flag 解析（confidence 不可信时不切状态）
        // CMMotionActivityConfidence 当前 SDK 只有 .low / .medium / .high；本实装把 .low 视作"低置信度"防抖入口.
        if activity.confidence == .low {
            return previous ?? .rest
        }

        // 规则 2-4：running > walking > stationary 优先级（多个 flag 同时 true 时按此顺序裁决）
        if activity.running {
            return .run
        }
        if activity.walking {
            return .walk
        }
        if activity.stationary {
            return .rest
        }

        // 规则 5：cycling / automotive / unknown / 全 false 兜底 → .rest
        // docs §10.2 "坐下按静止处理"同精神扩展：业务三态闭集，非 walk/run 即 rest.
        return .rest
    }
}
```

**关键决策与坑（必须严格遵守）**：

1. **pure function（struct/enum + static func）**：`MotionStateMapper` 用 `enum` 而非 `struct` 是 Swift 推荐的"namespace 用法"（无实例 / 不可继承 / 编译期更友好）。**禁止**改成 class 或 final class 持有 state；状态由 caller 持有传入。
2. **优先级 running > walking > stationary**：与 epics.md AC 行 1514 钦定严格对齐；**禁止**改成"按 flag 数量裁决" / "按 confidence 加权" / "按时间窗投票" 等复杂策略（YAGNI；docs §10.2 没要求；MVP 阶段一对一 mapping 足够）。
3. **confidence == .low 防抖路径**：`previous` 参数让 mapper 仍 pure（同输入同输出），caller 持有"上一次状态"在 ViewModel `@Published petState` 字段（Story 8.4 落地）。**禁止**用 `static var lastState: MotionState?` 这种全局可变状态——破坏 pure function 语义、跨 ViewModel 共享导致诡异 bug、单元测试需要手动 reset。
4. **previous default = nil + 兜底 .rest**：caller 首次调用没历史时直接传 nil；mapper 用 `previous ?? .rest` 兜底中性态。**禁止**首次调用兜底 .walk / .run（误导 UI 显示初始活动状态）。
5. **不抛错 / 不返回 Optional**：mapper 必返三态之一，输入永远有合法映射（活动 flag 全 false 也归 .rest）。**禁止**改成 `throws` 或 `MotionState?`——caller 不需要错误处理路径，简化上游链。
6. **不写 print / log**：pure function 零副作用；如需调试由 caller（Story 8.4 HomeViewModel.bind 闭包内）按需打 os_log。
7. **`@inlinable` / `@inline(__always)` 不加**：YAGNI；mapper 调用频率低（每次系统 push activity 1 次，频次 ≪ 1Hz），优化收益小于代码复杂度成本。
8. **`Sendable`-friendly**：mapper 是 enum 无实例，自然 Sendable；`map(_:previous:)` 接受 `CMMotionActivity`（非 Sendable）+ `MotionState`（Sendable enum），返回 `MotionState`——caller 在哪个 actor 调都安全（**入参 CMMotionActivity 跨 actor 传递的安全性由 caller 保证**——Story 8.2 `MotionProviderImpl` 已钦定 callback 在 OperationQueue.main 触发，Story 8.4 HomeViewModel.bind 在 main actor 接收，自然同 actor）.
9. **不写 `mapToString(_:)` 转字符串便利方法**：YAGNI；caller 用 `MotionState.rawValue` 即可拿到 "rest" / "walk" / "run"（AC1 钦定的 String raw value）；Story 8.5 `SyncStepsUseCase` 拼请求体直接走 `petState.rawValue`，不需要 mapper 提供 string helper。
10. **不写 `mapBatch([CMMotionActivity])`**：本 story 仅 single activity → state；批量映射（如 8.5 收集 5 分钟内多次 activity 决策最终态）由 caller 按需聚合（YAGNI）。

### AC3 — 单元测试 ≥6 case（`iphone/PetAppTests/Core/Motion/MotionStateMapperTests.swift`）

按 ADR-0002 §3.1 / §3.2 钦定 + epics.md AC 行 1509-1515 钦定 6 case（含 1 个 confidence 防抖 case）：

**新建文件**：`iphone/PetAppTests/Core/Motion/MotionStateMapperTests.swift`

```swift
// MotionStateMapperTests.swift
// Story 8.3 AC3: MotionStateMapper.map(_:previous:) pure function 单元测试.
//
// 测试用 Story 8.2 MotionProviderMock.makeActivity(...) 便利构造 CMMotionActivity 实例
// （MotionProviderMock 在 production target，PetAppTests 通过 @testable import PetApp 直接复用）.
// 不引第三方断言 lib（XCTest only；ADR-0002 §3.1）.

import XCTest
import CoreMotion
@testable import PetApp

@MainActor
final class MotionStateMapperTests: XCTestCase {
    // happy: stationary=true → .rest（epics.md AC 行 1510）
    func testMap_stationaryActivity_returnsRest() {
        let activity = MotionProviderMock.makeActivity(stationary: true)
        let result = MotionStateMapper.map(activity)
        XCTAssertEqual(result, .rest)
    }

    // happy: walking=true → .walk（epics.md AC 行 1511）
    func testMap_walkingActivity_returnsWalk() {
        let activity = MotionProviderMock.makeActivity(walking: true)
        let result = MotionStateMapper.map(activity)
        XCTAssertEqual(result, .walk)
    }

    // happy: running=true → .run（epics.md AC 行 1512）
    func testMap_runningActivity_returnsRun() {
        let activity = MotionProviderMock.makeActivity(running: true)
        let result = MotionStateMapper.map(activity)
        XCTAssertEqual(result, .run)
    }

    // happy: cycling=true（其他类型）→ .rest（epics.md AC 行 1513）
    func testMap_cyclingActivity_returnsRest() {
        let activity = MotionProviderMock.makeActivity(cycling: true)
        let result = MotionStateMapper.map(activity)
        XCTAssertEqual(result, .rest)
    }

    // edge: 多个 flag 同时为 true（walking + stationary）→ 优先级 run > walk > rest（epics.md AC 行 1514）
    // 验证 walking + stationary 同时 true 时 mapper 取 .walk（walking 优先于 stationary）
    func testMap_walkingPlusStationary_returnsWalkByPriority() {
        let activity = MotionProviderMock.makeActivity(stationary: true, walking: true)
        let result = MotionStateMapper.map(activity)
        XCTAssertEqual(result, .walk, "walking + stationary 同时为 true 时应按优先级返回 .walk")
    }

    // edge: 多个 flag 同时为 true（running + walking + stationary）→ 优先级 run > walk > rest（epics.md AC 行 1514）
    // 验证 running 优先于 walking 优先于 stationary
    func testMap_runningPlusWalkingPlusStationary_returnsRunByPriority() {
        let activity = MotionProviderMock.makeActivity(stationary: true, walking: true, running: true)
        let result = MotionStateMapper.map(activity)
        XCTAssertEqual(result, .run, "三 flag 同时为 true 时应按优先级返回 .run")
    }

    // edge: confidence=.low → 不切换状态，返回上一次值（epics.md AC 行 1515）
    // 防抖路径：低置信度 + previous=.walk → 保持 .walk（不被 .running flag 改写）
    func testMap_lowConfidence_withPreviousWalk_keepsPrevious() {
        let activity = MotionProviderMock.makeActivity(running: true, confidence: .low)
        let result = MotionStateMapper.map(activity, previous: .walk)
        XCTAssertEqual(result, .walk, "confidence=.low 时应保持上一次状态 .walk，忽略 running flag")
    }

    // edge: confidence=.low + previous=nil → 兜底 .rest
    // 首次调用低置信度路径的兜底中性态
    func testMap_lowConfidence_withNoPrevious_returnsRestAsFallback() {
        let activity = MotionProviderMock.makeActivity(walking: true, confidence: .low)
        let result = MotionStateMapper.map(activity)
        XCTAssertEqual(result, .rest, "confidence=.low + previous=nil 时应兜底 .rest")
    }

    // 加分项 case 9：unknown=true（兜底分支）→ .rest
    // 覆盖规则 5：非 stationary/walking/running/cycling/automotive 全 false → .rest
    func testMap_unknownActivity_returnsRest() {
        let activity = MotionProviderMock.makeActivity(unknown: true)
        let result = MotionStateMapper.map(activity)
        XCTAssertEqual(result, .rest)
    }

    // 加分项 case 10：confidence=.high + 多 flag → 优先级生效，不走防抖路径
    // 验证 confidence=.high（非 .low）时 previous 参数被忽略，按 flag 优先级裁决
    func testMap_highConfidence_withPreviousIgnored_returnsByFlagPriority() {
        let activity = MotionProviderMock.makeActivity(running: true, confidence: .high)
        let result = MotionStateMapper.map(activity, previous: .rest)
        XCTAssertEqual(result, .run, "confidence=.high 时 previous 应被忽略，按 flag 返回 .run")
    }
}
```

**关键决策**：

- **6 个 epics.md AC case 精确映射 + 4 个加分项（共 10 case）**：
  - case 1（stationary → .rest）→ AC 行 1510
  - case 2（walking → .walk）→ AC 行 1511
  - case 3（running → .run）→ AC 行 1512
  - case 4（cycling → .rest）→ AC 行 1513
  - case 5（walking + stationary 优先级）→ AC 行 1514
  - case 6（confidence=.low 防抖）→ AC 行 1515
  - 加分 case 7（running + walking + stationary 三 flag 优先级 .run）：补强 AC 行 1514 优先级最高优先级路径

### 来自 Story 8.2 review/dev 过程中的关键 lesson（与本 story 高度相关）

- **`docs/lessons/2026-05-04-motion-handler-invoke-must-be-in-lock.md`**：MotionProviderImpl 的 handler 调用必须与 generation check 共享同一锁段（避免 stop/restart race）。本 story mapper 是 pure function 无锁路径，但加深了"pure 与 stateful 边界要清晰"的理解—— mapper 不持任何状态（包括 lastState），状态由 caller 持有，mapper 同输入同输出.

## Latest Tech Information（CoreMotion + Swift 6.3 + Xcode 26）

- **`CMMotionActivity`** 字段：`stationary` / `walking` / `running` / `cycling` / `automotive` / `unknown`（Bool）+ `confidence`（CMMotionActivityConfidence: low / medium / high）+ `startDate`（Date）；多个 flag 可同时为 true（系统识别模糊时）.
- **`CMMotionActivityConfidence`** 是 enum 不是 Comparable — Swift 不自动派生 Comparable；本 story AC2 钦定用 `==` 判定（**禁止** `<` / `>` 比较，会编译错误）.
- **iOS 17+ deployment target** 完全覆盖 CMMotionActivityConfidence 的 .low / .medium / .high 三档；ADR-0002 §4 锁 deployment iOS 17 充足.
- **Swift 6.3 strict concurrency** 下 enum + static func 是天然 Sendable；mapper 无需 `@MainActor` / `actor` 隔离.
- **CMMotionActivity 不是 Sendable** — 但本 story mapper `static func` 接收 CMMotionActivity 作为参数；caller（Story 8.4 HomeViewModel.bind closure）调用 mapper 在哪个 actor 取决于 8.4 实装：8.2 钦定 callback 在 OperationQueue.main 触发 → 8.4 在 main actor 接收 → mapper 调用也在 main actor，自然安全.
- **CMMotionActivity 默认 init**（`CMMotionActivity()`）所有 Bool flag = false / confidence = .low — 测试中**禁止**用此默认值构造（详 8.2 Debug Log 关于 KVC crash 的 lesson；改用 `MotionProviderMock.makeActivity(...)`）.
